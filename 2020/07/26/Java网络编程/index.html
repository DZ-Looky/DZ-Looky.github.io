API调用

API

全称Application Programming Interface，应用程序接口，一般是指一些预先定义的函数，方便开发人员快速访问某一程序而无需了解其源码及内部工作细节。API可以快速调用某个程序

依赖库：

文件pom.xml中唯一的<dependencies>标签中，包含多个<dependency>标签，每个标签表示一个依赖库。此文件的作用就是定义此Java项目需要用到哪些库

Okhttp3页面请求

  1. 实例化OkHttpClient。OkHttpClient okHttpClient = new OkHttpClient();
  2. 执行调用
    1. 执行调用前需要实例化一个Request对象，用来定义请求的各种参数。Request request = new Request.Builder().url(url).build();
    2. 然后构建调用对象。Call call = okHttpClient.newCall(request);
    3. 最后执行调用。如果调用失败可能抛出异常。所以必须抓取异常。call.execute()
  3. call.execute()返回一个执行的结果对象，通过方法获取返回的字符串内容:call.execute().body().string();

    以上代码基本属于固定写法

解析JSON对象

查询IP地址信息：
Map contentObj = JSON.parseObject(content, Map.class);
System.out.println("当前 IP 为：" + contentObj.get("query")));

如果是复杂的多层嵌套的JSON数据，则多次取出嵌套的Map对象

Map contentObj = JSON.parseObject(content, Map.class);
Map dataObj = (Map)contentObj.get("data");

因为Map可以储存任何对象，所以从Map中get()道德对象必须指定其实际的类型: (Map)、(String)

User-Agent

User-Agent是存放在Headers中的一种数据信息。HTTP消息头Headers

Okhttp3库添加Headers：构建Request对象时调用addHeader()方法

Referer

有时一些网站会将图片数据做“防盗链”处理，即通过其他网站访问此图片时不能被访问，只有此图片原本自身网站访问才被允许。而消息头中的Referer信息是浏览器在请求网站中的图片或其他文件时自动添加在Headers中的信息，用来表示请求的来源。

因此，我们需要手动添加Referer信息为所要访问文件的原始网站。

Session

cookie是存放在客户端浏览器的,且是临时的,过一段时间后会失效.而使用Session对象可以解决这个问题

收集信息打开谷歌浏览器开发者工具,勾选Preserve log,输入用户名密码进行登录,再从右侧抓取的网络请求中找到登录请求.

需要收集的关键信息包括：

  1. 登录地址
  2. 执行方式（GET/POST）
  3. HOST
  4. Referer
  5. User-Agent
  6. 表单数据，数据名称和数据值

为了能够复用session，我们将OkHttpClient对象进行重构,不再定义为postContent()方法的变量,而改为类变量private static final OkHttpClient okHttpClient,使得能够在整个类中可以使用同一个okHttpClient多次执行Http请求.

将执行请求的过程分为两个方法过程,提交表单进行登录和获取返回数据的方法执行过程都是类似的,区别是Request不同,因此可以把真正执行的过程单独封装为一个doExecute()方法,通过传入其他两个方法获取的Request得到不同的结果
