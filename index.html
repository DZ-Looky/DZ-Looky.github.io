API

全称Application Programming Interface，应用程序接口，一般是指一些预先定义的函数，方便开发人员快速访问某一程序而无需了解其源码及内部工作细节。API可以快速调用某个程序

文件pom.xml中唯一的<dependencies>标签中，包含多个<dependency>标签，每个标签表示一个依赖库。此文件的作用就是定义此Java项目需要用到哪些库

Okhttp3页面请求

  1. 实例化OkHttpClient。OkHttpClient okHttpClient = new OkHttpClient();
  2. 执行调用
    1. 执行调用前需要实例化一个Request对象，用来定义请求的各种参数。Request request = new Request.Builder().url(url).build();
    2. 然后构建调用对象。Call call = okHttpClient.newCall(request);
    3. 最后执行调用。如果调用失败可能抛出异常。所以必须抓取异常。call.execute()
  3. call.execute()返回一个执行的结果对象，通过方法获取返回的字符串内容:call.execute().body().string();

User-Agent

User-Agent是存放在Headers中的一种数据信息。HTTP消息头Headers

Referer

有时一些网站会将图片数据做“防盗链”处理，即通过其他网站访问此图片时不能被访问，只有此图片原本自身网站访问才被允许。而消息头中的Referer信息是浏览器在请求网站中的图片或其他文件时自动添加在Headers中的信息，用来表示请求的来源。

因此，我们需要手动添加Referer信息为所要访问文件的原始网站。

接口

Posted on 2020-07-14

接口

public interface xxxService {}
一般情况下我们定义接口的名称遵循xxxService,以Service结尾的类名称代表的就是接口

  • 接口的方法不需要方法体，故不能存在大括号
  • 接口的方法都是public

接口实现类

  • 接口定义的方法在实现类必须全部实现，而且方法名称、参数、返回值必须完全相同
  • 由于接口方法都是public的，所以实现类的方法也必须为public

实现类实现多个接口时用,分开即可

实例化接口

接口只是定义，没有具体实现，故
EchoService echo = new EchoService();是错误的
正确的实例化应该结合实现类
EchoService echo = new EchoServiceImpl();

继承

方法覆盖
通过子类对象实例化父类对象时，调用同名方法时会因为同名覆盖的原因执行子类方法

super

  • 子类调用父类同名方法
  • 子类实现父类构造函数
  • 当父类只有一个有参数的构造函数的时候，子类也必须要具备这个构造函数，或者调用 super 方法为父类构造函数提供参数来实现新的构造函数
  • 构造函数里super方法必须写在第一行
